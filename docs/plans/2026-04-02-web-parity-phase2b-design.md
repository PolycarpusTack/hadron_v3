**Date:** 2026-04-02
**Status:** Approved

Embedding-based duplicate ticket detection and JIRA comment posting with engineer feedback.

## Backend

### Duplicate Detection

**Reuse existing infrastructure.** The web already has:
- `embeddings` table with pgvector (migration 008) — `source_type`, `source_id`, `embedding vector(1536)`, `content`
- `integrations/embeddings.rs` — `generate_embedding(text, api_key)` calls OpenAI `text-embedding-3-small`
- `db/mod.rs` — `store_embedding()`, `find_similar_analyses()` with pgvector `<=>` operator

**Approach:** Store ticket embeddings in the existing `embeddings` table with `source_type = 'ticket'`. Since `source_id` is `BIGINT` and jira_keys are strings, use a deterministic hash: `source_id = hash(jira_key) as i64`.

New DB functions:
- `store_ticket_embedding(pool, jira_key, embedding, content)` — stores with `source_type = 'ticket'`, `source_id = hash(jira_key)`
- `find_similar_tickets(pool, embedding, exclude_key, threshold, limit) -> Vec<SimilarTicketMatch>` — joins `embeddings` (source_type='ticket') with `ticket_briefs` on hashed key, uses pgvector cosine distance, returns matches above threshold

```rust
pub struct SimilarTicketMatch {
    pub jira_key: String,
    pub title: String,
    pub similarity: f64,
    pub severity: Option<String>,
    pub category: Option<String>,
}
```

Embedding text construction (`build_ticket_embedding_text`):
- If brief_json exists: extract `analysis.plain_summary` + `analysis.technical.root_cause` + `triage.customer_impact`
- Fallback: `title + "\n\n" + description`

New route:
- `POST /api/jira/issues/{key}/similar` — request: `{ credentials, threshold?, limit? }`. Fetches or generates embedding on-the-fly, runs similarity search, returns `Vec<SimilarTicketMatch>`.

Fire-and-forget embedding generation after brief creation (add to existing `generate_brief` route).

### JIRA Round-Trip

**JIRA comment posting** — add to `integrations/jira.rs`:
- `post_jira_comment(config, key, body) -> HadronResult<()>` — `POST /rest/api/2/issue/{key}/comment` with `{ "body": text }`
- `format_brief_as_jira_markup(brief: &JiraBriefResult, jira_key: &str) -> String` — formats as JIRA wiki markup:

```
h3. Hadron Investigation Brief — {KEY}

*Severity:* {severity} | *Category:* {category} | *Confidence:* {confidence}

h4. Summary
{plain_summary}

h4. Root Cause
{root_cause}

h4. Recommended Actions
* *[{priority}]* {action} — _{rationale}_

*Risk:* {blast_radius} blast radius, {urgency} urgency. {do_nothing_risk}

----
_Generated by Hadron JIRA Assist_
```

**DB functions:**
- `mark_posted_to_jira(pool, jira_key)` — sets `posted_to_jira = true`, `posted_at = NOW()`
- `update_engineer_feedback(pool, jira_key, rating, notes)` — updates `engineer_rating` and `engineer_notes`

**Routes:**
- `POST /api/jira/issues/{key}/post-brief` — request: `{ credentials }`. Loads brief from DB, formats as markup, posts comment, marks posted.
- `PUT /api/jira/briefs/{key}/feedback` — request: `{ rating?, notes? }`. Updates feedback fields.

### TicketBriefPanel extensions

**Props changes:** Add `jiraCredentials: JiraCredentials`, `onBriefUpdated?: () => void` (callback to refresh cached brief after post/feedback).

**New sections/buttons:**

1. **"Similar Tickets"** section at bottom of Brief tab:
   - "Find Similar" button, loading state
   - Results: cards with jira_key (monospace), title, similarity bar (width proportional to %), severity/category badges
   - Threshold 0.65, limit 5
   - Empty state: "No similar tickets found"

2. **"Post to JIRA"** button in header:
   - `window.confirm("Post this brief as a comment on {key}?")`
   - On success: show "Posted" green badge + timestamp, disable button
   - Disabled if `briefRow?.postedToJira` is true (load from parent)

3. **Engineer feedback** in header:
   - Star rating: 5 stars, click to set, golden filled stars, saves via `submitEngineerFeedback()` on click
   - Notes: small textarea toggle, saves on blur via `submitEngineerFeedback()`
   - Initialize from `briefRow?.engineerRating` and `briefRow?.engineerNotes`

### JiraAnalyzerView changes

Pass JIRA credentials and `onBriefUpdated` callback to TicketBriefPanel:
```tsx
<TicketBriefPanel
  jiraKey={ticket.key}
  result={briefResult}
  jiraCredentials={{ baseUrl, email, apiToken }}
  briefRow={cachedBrief}
  onBriefUpdated={async () => {
    const updated = await api.getTicketBrief(ticket.key);
    setCachedBrief(updated);
  }}
/>
```
