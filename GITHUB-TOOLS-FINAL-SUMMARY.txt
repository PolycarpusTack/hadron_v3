CONCRETE EXAMPLE: WHY WINSTON MATTERS

Current code (src/services/circuit-breaker.ts, line 47):
  console.warn(`🔴 Circuit breaker OPENED for ${provider}`);
  
Problem in production:
  - No timestamp
  - No context (which provider? what error rate?)
  - No way to find this in logs (if file exists)
  - Can't search or aggregate

With Winston:
  logger.warn('circuit_breaker_opened', {
    provider: 'openai',
    errorRate: 0.65,
    timestamp: '2025-11-13 14:30:45',
  });

Result:
  - Structured JSON in hadron.log
  - Can search/filter by provider
  - Aggregatable to monitoring systems
  - Production visibility

CONCRETE EXAMPLE: WHY PLAYWRIGHT MATTERS

Current testing:
  - You manually click buttons
  - You manually watch results
  - Every code change requires manual re-testing
  - Human error: miss edge cases

With Playwright:

    await page.goto('http://localhost:5173');
    await page.click('button:has-text("Choose File")');
    await page.setInputFiles('input[type="file"]', 'crash.log');
    await page.waitForText('Analysis Results');
    await expect(page.locator('text=ROOT CAUSE')).toBeVisible();
  });

Result:
  - Runs 30 seconds automatically
  - Runs every commit
  - Catches regressions immediately
  - Confidence to ship
