# Ask Hadron 2.0 — Knowledge Flywheel Design

**Date:** 2026-02-18
**Approach:** Knowledge Flywheel — gold verification, knowledge capture, and retrieval loop with comprehensive UX enhancements
**Audience:** Support engineers and developers equally
**Strategy:** Complete package (cohesive overhaul)

---

## 1. Gold Answer System (Flywheel Core)

When a user gets a great answer, they mark it as "gold" — a verified, trustworthy Q&A pair that becomes retrievable context for future questions.

### Data Model

New `gold_answers` table:

| Column | Type | Purpose |
|---|---|---|
| `id` | INTEGER PK | Auto-increment |
| `question` | TEXT | The user's original question |
| `answer` | TEXT | The verified assistant response |
| `session_id` | TEXT | Link to chat session |
| `message_id` | TEXT | The specific assistant message |
| `won_version` | TEXT NULL | WON version context if set |
| `customer` | TEXT NULL | Customer context if set |
| `tags` | TEXT NULL | Comma-separated tags |
| `verified_by` | TEXT NULL | Who marked it gold |
| `tool_results_json` | TEXT NULL | Serialized tool results for provenance |
| `created_at` | TEXT | Timestamp |

### UX Flow

1. User sees a good answer and clicks a star/badge icon on the message.
2. A dialog appears: "Save as Gold Answer" with optional tag input (e.g., `scheduling`, `PSI`, `Oracle`, `VRT`).
3. On confirm, the Q&A pair + metadata is stored in `gold_answers`.
4. The message gets a visible gold badge.

### Retrieval Integration

- New tool: `search_gold_answers` — searches gold_answers by text similarity.
- System prompt updated to try gold answers first.
- Gold answers appear with a "Verified Answer" badge in responses.
- Gold answers are local-first (SQLite), not pushed to OpenSearch.

## 2. Conversation Summaries

One-click generation of a polished, structured summary from any chat session for sharing, indexing, and documenting resolved issues.

### How It Works

1. User clicks "Summarize" in the session header or sidebar context menu.
2. Backend sends the full conversation to the LLM with a dedicated summarization prompt.
3. Summary is streamed back and displayed in a modal/panel.
4. User can edit the summary before saving.
5. On save, stored alongside the session.

### Summary Structure

```
## Topic
[One-line description]

## Context
- WON Version: {version}
- Customer: {customer}
- Related Analyses: {#IDs}

## Question
[Core question or problem]

## Answer
[Condensed, verified answer]

## Sources
[KB docs, analysis IDs, JIRA tickets cited]

## Resolution
[Action taken or recommended]
```

### Data Model

New `session_summaries` table:

| Column | Type | Purpose |
|---|---|---|
| `id` | INTEGER PK | Auto-increment |
| `session_id` | TEXT UNIQUE | Link to chat session |
| `summary_markdown` | TEXT | The edited/approved summary |
| `topic` | TEXT | Extracted topic line |
| `won_version` | TEXT NULL | Version context |
| `customer` | TEXT NULL | Customer context |
| `is_indexed` | BOOLEAN | Exported for RAG indexing |
| `created_at` | TEXT | Timestamp |
| `updated_at` | TEXT | Last edit |

### Export Options

- **Copy to clipboard** (plain text, markdown stripped) — for JIRA, email, Slack.
- **Export as .txt** — plain text file, headers as UPPERCASE labels.
- **Export as .md** — markdown for documentation/KB indexing.
- **Insert as JIRA comment** — posts directly to a user-specified ticket via existing JIRA integration.

## 3. Fine-tuning Export Pipeline

Exports gold Q&A pairs and summaries for model fine-tuning (LoRA) and RAG ingestion.

### Export Formats

**JSONL for fine-tuning:**
```jsonl
{"messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]}
```
- Includes system prompt context for tone/citation style learning.
- `_metadata` field for filtering (ignored by training).
- User chooses filters: date range, customer, tags, WON version.

**Markdown bundle for RAG indexing:**
```
export/
  summaries/
    2026-02-18-psi-namespace-conflict.md
    ...
  metadata.json
```
Individual `.md` files per summary with a metadata index.

### UX

- Export section in Settings or sidebar button.
- "Export Gold Answers" -> file picker -> `.jsonl`.
- "Export Summaries for RAG" -> folder picker -> markdown bundle.
- Shows item count and date range before export.
- Items marked `is_exported = true` after export (user can override with "export all").
- Export is always manual — no auto-sync.

## 4. Session Management

Makes the session sidebar a proper workspace.

### Features

**Search:** Input at sidebar top. Searches session titles and summary topics. Real-time client-side filtering.

**Stars & Tags:**
- Sessions can be starred/pinned (stick to top of list).
- Sessions can be tagged with labels (same tag system as gold answers).
- Tags are clickable filters.

**Filter bar:** Compact row below search:
- By customer (dropdown)
- By WON version
- By status: All / Starred / Has Summary / Has Gold Answers
- Date range: 7d / 30d / 90d / all

**Session row display:**
- Title, timestamp, customer badge, icons for starred/summary/gold.
- Right-click context menu: Rename, Star/Unstar, Tag, Summarize, Export, Delete.

### Data Model Changes

Add to existing `chat_sessions` table:
- `is_starred` BOOLEAN DEFAULT 0
- `tags` TEXT NULL
- `customer` TEXT NULL
- `won_version` TEXT NULL

All filtering is client-side on loaded session metadata.

## 5. Export & Sharing

Gets content out of Ask Hadron in useful formats.

### Session-Level Export

Button in session header:
- **Markdown (.md)** — full conversation, messages labeled `**You:**` / `**Hadron:**`.
- **Plain text (.txt)** — clean text, no formatting.
- **Interactive HTML (.html)** — styled standalone page with collapsible messages and source links.

### Message-Level Actions

Per-message context:
- **Copy formatted** — rich text (markdown rendered) for Slack/Teams/email.
- **Copy plain text** — stripped markdown.
- **Copy as quote** — blockquote with attribution: `> [Ask Hadron, 2026-02-18] ...`

### Summary Export

As defined in Section 2: clipboard, .txt, .md, JIRA comment.

All export is local files or clipboard. No cloud sharing service.

## 6. Response Control

User control over answer generation without starting new conversations.

### Regenerate

- Refresh icon on the last assistant message.
- Re-sends the same user message with same context.
- Replaces the old response (no alternatives kept).

### Edit & Re-ask

- Pencil icon on user messages.
- Loads message content into input textarea.
- Conversation rewound to that point (later messages removed).
- User edits and sends for a fresh response.

### Verbosity Control

- Toggle in input area: Concise / Detailed.
- Concise: appends "Be brief. 2-3 sentences max." to system prompt.
- Detailed: appends "Be thorough. Include all relevant details, examples, and source citations."
- Default: neither (current 2-3 paragraph behavior).
- Persists per session.

No mid-conversation model switching. Model is set globally in Settings.

## 7. Tool Transparency

Shows exactly what the agent did behind the scenes.

### Expanded Tool Trace

Replaces the minimal DiagnosticsPanel. Each tool call is an expandable card:
- **Header:** tool name + one-line summary (e.g., `search_kb -> "PSI namespace" -> 8 results`)
- **Expanded:** arguments passed, first 2-3 results with truncated content, timing per tool call.

### Source Panel

- Existing source badges (KB: 5, Analyses: 3) become clickable.
- Opens a slide-out panel listing all sources for that response.
- Each source: title, clickable URL, source type badge, relevance score.
- KB docs show the extracted text snippet fed to the LLM.

### Token & Cost Display

- Per-message footer: `847 tokens . $0.003`
- Session total in header: `Session: 12,450 tokens . $0.05`

### Evidence Assessment (Enhanced)

- Insufficient evidence: visible amber banner above response with suggestion to refine.
- Sufficient evidence: subtle green dot next to response.
